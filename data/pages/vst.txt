====== VST (Virtual Studio Technology) ======

Le **VST** est un protocole et format de plug-in audio propriétaire dont le kit de développement appartient à la société **Steinberg**.

Il s'agit d'un format de **plugin** qui permet le développement aussi bien d'effets que d'instruments, qu'on peut utiliser dans des [[:mao#daws_studios_virtuels_et_sequenceurs_midi|studios virtuels]] qui seront leurs "**hôtes**".

**[[http://linuxmao.org/Vestige|VeSTige]]**, un projet lié à [[:LMMS]], a permis de fournir une implémentation ouverte de **VST** et donc d'utiliser le protocole dans le cadre de projets libres.\\ 
Grâce à cela ces dernières années ont vu l'émergence de nombreuses applications implémentant ce protocole au sein de l'univers Gnu/Linux. De nombreux plugins ont pu être portés ou développés nativement pour Linux.\\ 
**[[http://www.joebutton.co.uk/fst/|FST]]** permet même (grâce à [[:Wine]]) de charger des plugins compilés pour Windows (extension //.dll//) dans des logiciels Linux.

===== Plugins =====

==== Plugins natifs Linux ====

Une liste impressionnante de plugins compilés pour Linux sur cette page : **(en)** http://linux-sound.org/linux-vst-plugins.html

==== Plugins Windows ====

Il est possible d'utiliser la plupart des VST compilés pour Windows (au format DLL) dans Linux (grâce à [[http://www.joebutton.co.uk/fst/|FST]]) :

[[http://festige.sourceforge.net|FeSTige]] permet de les inclure comme plugin [[:DSSI]], ou comme standalone [[:jackd|JACK]].

[[http://kxstudio.linuxaudio.org/Applications:Carla|Carla]] permet également, grâce à [[https://linuxmusicians.com/viewtopic.php?f=47&t=13280|carla-bridge-win32]] de les utiliser en standalone [[:jackd|JACK]], ou en plugin [[:LV2]] ou VST (dans un hôte qui attend du natif Linux).

En plus des "adaptateurs" comme FeSTige et Carla, certains studios acceptent directement des plugins compilés pour Windows :
  * [[:LMMS]] (via [[apt>lmms-vst-full]] dispo dans les [[http://kxstudio.linuxaudio.org/Repositories|dépots KXStudio]])
