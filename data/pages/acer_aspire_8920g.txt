===== Problèmes/Conseils liés à l'installation =====
==== Si vous n'arrivez pas à démarrer sur le Desktop-CD ====

Il faut modifier l'ordre de démarrage du BIOS : pour cela démarrer sous Vista puis lancer le logiciel 'Empowering Technologie'. Ce logiciel est préinstallé sur Vista. Cliquer sur l'icône avec une roue dentée. Puis dans l'onglet Options de démarrage, sélectionner Lecteur CD-ROM/DVD, puis avec la flèche faites le monter au plus haut.
Tant que l'on y est, décochez l'option démarrage rapide. En effet, si vous avez décidé d'ajouter une entrée dans Bootmgr (cf : http://doc.ubuntu-fr.org/tutoriel/comment_amorcer_ubuntu_avec_bootmgr) l'option démarrage rapide n'affichera pas le menu qui permet de choisir son système d'exploitation.
Redémarrer ensuite sur votre CD.

==== Partitions ====

Par défaut, vous devez avoir 4 partitions primaires sur votre Disque Dur. Ce qui est embêtant car on est limité à 4. Il faut donc en supprimer deux pour installer Ubuntu. J'ai supprimé la DATA et la PQSERVICE.

<note warning>
Si vous supprimez la partitions DATA assurez vous bien que vous n'avez pas de données sinon elles seront perdues ...
</note>

<note important>
La partition PQSERVICE sert normalement à la restauration de Windows. Chez moi, Vista fonctionne sans cette partition. Attention, assurez vous d'avoir gravé vos CD/DVD de restauration usine avant de supprimer cette partition.
Tout ce protocole marche chez moi, mais ça reste à vos risque et périls, et personne de Ubuntu-fr ne pourra être tenu pour responsable de la perte de données ou autre ...
</note>

==== Partition étendue ====

Lors d'une première installation, j'ai essayé de créer une partition étendue, mais à chaque fois il semble qu'elle soit corrompue par Windows. C'est pour cela que j'ai créé deux partitions primaires pour Ubuntu.
J'ai fait une partition Swap-Linux et une partition / en ext3.

<note>En fait, j'ai récemment trouvé mieux pour repartir d'une configuration propre : réinstaller complètement Windows. Graver vos CD de réinstallation (configuration usine) sous Vista, puis démarrer sur le CD d'Ubuntu et avec le logiciel Gparted créer une nouvelle table de partitions //msdos// (//Attention// cela efface **toutes** vos donnés sur le Disque Dur : faites des sauvegardes sur un périphérique externe (clef USB, CD, Disque Dur externe ...)). Puis créer toutes vos partitions dont une pour Vista en NTFS. Ci-dessous voila ce que ça donne (je vous conseille de faire comme moi). Ensuite mettre dans le lecteur le 1er CD (ou DVD) de réinstallation de Vista (Vista s'installera sur la partion en NTFS sans toucher aux autres ce qui nous arrange !). Enfin quand Vista est installé on peut installer Ubuntu.</note>

==== Le son ====

=== Pour Jaunty Jackalope (9.04) ===

Il n'est donc pas possible de faire fonctionner le Surround ? 
Si ! sous jaunty (pas testé sur les autres versions) il est possible de faire fonctionner le surround 5.1. Voici la manipulation : [[:tutoriel:comment_modifier_un_fichier|ouvrez le fichier]] **/etc/modprobe.d/alsa-base.conf** avec les droits d'administration.

Entrez votre mot de passe, puis ajoutez ces trois lignes à la fin du texte et sauvegardez :

<file>
# added by myself

alias snd-card-0 snd-hda-intel

options snd-hda-intel model=auto
</file>

Puis [[:tutoriel:comment_modifier_un_fichier|ouvrez le fichier]] **/etc/pulse/daemon.conf**.
Cherchez la ligne (( Pour chercher une ligne particulière dans gedit, on peut utiliser le menu : Rechercher=>Rechercher... )):

<file>; default-sample-channels = 2</file>

Et remplacez le 2 par un 6 (pour le 5.1) et décommentez-le ; ce qui donne :

<file>default-sample-channels = 6</file>

=== Pour Intrepid Ibex (8.10) ===

Pour faire marcher les 2 speakers avant en stéréo et le casque. [[:tutoriel:comment_modifier_un_fichier|Ouvrez le fichier]] **/etc/modprobe.d/alsa-base** avec les droits d'administration puis ajoutez ces deux lignes a la fin du texte et sauvegardez :

<file>
# added by myself

alias snd-card-0 snd-hda-intel

options snd-hda-intel model=auto
</file>

Redémarrez pour que ça marche.
