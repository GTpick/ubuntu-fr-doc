===== Caractéristiques =====

  * **Mémoire :** 1Go DDR2 ;
  * **Ethernet : ** Atheros AR8132 ;
  * **Sans fil :** 
    * Wifi : Atheros AR9285,
    * Bluetooth : 2.1 EDR,
    * 3G+ : Huawei E620 ou EM770 (en option),
  * **Carte vidéo :** Intel® 950 ;
  * **Carte son :** Intel® 82801G ;
  * **Webcam :** 0,3 ou 1,3 Mégapixels ;
  * **Disque dur :** (selon modèle et date de sortie) 
    * 160 Go,
    * 250 Go,
    * 320 Go,
  * **Écran :**
    * Diagonale : 10,1 pouces,
    * Résolution : 1024x600,
  * **Périphériques de sorties :**
    * 3 port USB,
    * 1 port VGA,
    * entrée/sortie audio
    * Lecteur de Cartes mémoires SD/SDHC.

=== Cliquetis du disque dur ===

En fonctionnement sur batterie, le disque dur fait entendre à chaque redémarrage un cliquetis. Si ces cliquetis sont très fréquents (cela peut atteindre toutes les 5 secondes), il faut augmenter le temps avant l'arrêt du disque dur en modifiant les paramètres de sa gestion.

En pratique, il suffit de recopier dans un terminal la commande suivante :

<code>cmd='hdparm -B 254 /dev/sda';sudo $cmd && echo -e '#!/bin/sh\n'"$cmd" > /tmp/hdfix && sudo install /tmp/hdfix /etc/pm/power.d/99-hdparm.sh && sudo install /tmp/hdfix /etc/pm/sleep.d/99-hdparm.sh</code>

Pour supprimer les effets de cette méthode :
<code>sudo rm /etc/pm/power.d/99-hdparm.sh /etc/pm/sleep.d/99-hdparm.sh</code>


=== Désactiver le touchpad ===

La combinaison des touches Fonction-F3 ne permet pas de désactiver le touchpad. Pour commander l'arrêt ou la marche du touchpad par les combinaisons windows-F3 et windows-F4, il suffit de modier le fichier : ~/.config/openbox/lxde-rc.xml . Il faut rechercher les lignes <keybind key="W-F3"> et <keybind key="W-F4"> et modifier ainsi les paragraphes :  
 
     <file>  <keybind key="W-F3">
      <action name="Execute">
	   <command>synclient TouchpadOff=1</command>
      </action>
    </keybind>
    <keybind key="W-F4">
      <action name="Execute">
	   <command>synclient TouchpadOff=0</command>
      </action>
    </keybind></file>
   
===== Optimisations =====

<note>**En l'absence d'optimisations spécifique à ce modèle**, veuillez consulter ceux indiqués sur la **documentation générale** des **[[:netbook|netbooks]]** et des **[[:asus_eee_pc|Eee PC]]**.</note>

Il est possible que dans certains cas, le pilote ath9k provoque une instabilité du débit (fluctuations, freezes en téléchargement, ou perte de connexion) lorsque l'ordinateur est connecté en Wifi. Dans ce cas, il suffit de procéder de la manière suivante pour résoudre le problème (adapté de la solution trouvée ici : http://ubuntuforums.org/showthread.php?t=1740056) :
<code>
sudo gedit /etc/modprobe.d/ath9k.conf
</code>
ou selon les préférences, dans le terminal (appuyer sur la touche "Insert" pour passer en mode insertion dans VIM)
<code>
sudo vim /etc/modprobe.d/ath9k.conf
</code>
Puis dans le fichier (normalement vide), ajouter la ligne suivante :
<code>
options ath9k nohwcrypt=1
</code>
Enregistrer le fichier (pour VIM : touche échap, puis :wq), et enfin, redémarrer l'ordinateur. Le wifi devrait fonctionner normalement. Testé sous Natty, avec succès.
