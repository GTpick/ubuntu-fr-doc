====== Gm-notify ======

<note important>Cette application n'est plus disponible dans les dépôts depuis Ubuntu 16.04. Reportez vous à la page [[:webmail#gmail|suivante]] pour une alternative.</note>

**gm-notify** est un programme de notification de réception de mails pour le service Gmail de Google utilisant libnotify-bin.\\
Il se veut léger et s'intègre parfaitement à indicator-messages (l'applet de notification de messages qui est installé par défaut sur Ubuntu) se situant sur le tableau de bord supérieur (voir capture d'écran ci-après).

{{gm-notify_natty_01.png}}

Les notifications d'arrivée se font en direct (Push mail), il n'y a donc pas de temps d'attente entre la réception du mail et la notification d'arrivée sur votre bureau.

Le projet est hébergé sur la plateforme [[Launchpad]] de Canonical (voir liens en bas de page).


===== Pré-requis =====

  * Disposer des [[:sudo|droits d'administration]].
  * Disposer d'une connexion à Internet configurée et activée.
 

===== Installation =====

Il suffit d'[[:tutoriel:comment_installer_un_paquet|installer le paquet]] **[[apt>gm-notify|gm-notify]]**.


===== Configuration =====

Lors du premier lancement, le programme devrait vous demander vos informations de connexion. Si ce n'est pas le cas, lancez l'utilitaire de configuration se trouvant dans le menu //Système -> Préférences -> GMail Notifier Configuration//. Cette fenêtre peut aussi être obtenue à l'aide d'un clic droit sur le menu « Google Mail » dans la zone de notification.

Enfin, vous pouvez aussi exécuter la [[:commande_shell|commande]] suivante dans un [[:terminal]]:
<code>gm-notify-config</code>

===== Utilisation=====

Vous pouvez lancez l'application directement depuis la zone de notification (voir capture d'écran ci-dessus) ou //via// la [[:commande_shell|commande]] suivante :
<code>gm-notify</code>

===== Désinstallation =====

Pour supprimer cette application, il suffit de [[:tutoriel:comment_supprimer_un_paquet|supprimer son paquet]]. Selon la méthode choisie, la configuration globale de l'application est conservée ou supprimée. Les journaux du système, et les fichiers de préférence des utilisateurs dans leurs dossiers personnels sont toujours conservés.
