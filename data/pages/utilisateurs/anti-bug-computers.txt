==== Sous 12.10 ====
=== Carte graphique NVidia ===
Nota : la génération GForce 6 ne fonctionne plus correctement avec la 12.10 (limité). \\
Votre carte graphique n'est pas forcément bien reconnu et Ubuntu permet une définition Max de 1024 x 768 avec le pilote générique. \\
 \\
Cette procédure à été testé avec une carte NVidia GeForce 210. Cependant elle est valable pour toutes les cartes récentes (voir la liste des cartes supportées sur la page du pilote nvidia-304-60 sur le site NVidia [[http://www.nvidia.fr/object/linux-display-amd64-304.60-driver-fr.html]]). \\
Attention ! : Il est a noter que dés que le noyau est mis à jour, il faut recommencer la procédure et ce, jusqu'a ce que le nouveau pilote NVidia soit intégré aux dépôts officiels. \\
 \\
Ouvrez votre terminal avec les touches  Ctrl  +  Alt  +  T , puis tapez la commande suivante : \\
<code>cd && wget http://dl.dropbox.com/u/47950494/upubuntu.com/nvidia-304-60</code>
fermer la session puis, \\
Appuyez sur  CTRL + ALT + F3 , connectez-vous avec votre identifiant et votre mot de passe et entrez la commande suivante pour tuer le serveur X: \\
<code>sudo service lightdm stop</code> POUR UBUNTU 12.10 (version normal) \\
ou \\
<code>sudo service gdm stop</code> POUR UBUNTU 12.10 Gnome Remix \\
Appuyez de nouveau sur  CTRL + ALT + F3 \\
Ensuite, démarrez l'installation pilote en tapant: \\
<code>cd && sudo sh nvidia-304-60</code> \\
Suivez les instructions puis lorsque c'est terminé, tapez ceci pour redémarrer votre ordinateur : \\
<code>sudo reboot</code>
Normalement vous devriez avoir la définition maximum de votre écran. \\
RAPPEL : Attention ! : Il est a noter que dés que le noyau est mis à jour, il faut recommencer la procédure et ce, jusqu'à ce que le nouveau pilote NVidia soit intégré aux dépôts officiels. \\
NOTA : Il semblerai que depuis le noyau Noyau Linux 3.5.0-17-generic , Ubuntu permet une définition d'écran de 1680 x 1050 (16:10) max pour un écran de 22" standard avec cette carte graphique par défaut (sans faire les manipulations ci-dessus), ce est acceptable pour cette taille d'écran  \\

=== Calibrez un écran tactile e-Galax ===
Dans le terminal, entrez la commande : \\
<code>lsusb</code>
si le résultat est le suivant : \\
<code>Bus 002 Device 002: ID 0eef:0001 D-WAV Scientific Co., Ltd eGalax TouchScreen</code>
Vous êtes en présence d'un écran tactile eGalax. \\
Pour calibrer finement cette écran tactile, installer ce paquet : [[apt://xinput-calibrator]] \\
puis utiliser le logiciel installer pour calibrer l 'écran \\
Une fois l'écran calibré, le logiciel donne les réglages qu'il faut intégré au fichier de configuration afin que ce calibrage soit définitif comme cette exemple :  \\
<code>  copy the snippet below into '/etc/X11/xorg.conf.d/99-calibration.conf'
Section "InputClass"
	Identifier	"calibration"
	MatchProduct	"eGalax Inc. Touch"
	Option	"Calibration"	"1982 21 165 1874"
EndSection</code>
Il faut copier la section dans le fichier : /etc/X11/xorg.conf (et pas dans  /etc/X11/xorg.conf.d/99-calibration.conf comme le document le suggère.)  \\
<code>sudo gedit /etc/X11/xorg.conf</code>
enregistrer et redémarrer.

Dans le terminal, coller ceci : lsusb \\
si le résultat est (modèle Vigogne Ivy-Bridge fin 2012 par exemple) : \\
<code>Bus 003 Device 002: ID 0bda:8723 Realtek Semiconductor Corp.</code>
Ouvrir un terminal et installer d'abord les paquets nécessaires à la compilation, via cette ligne de commande : \\
<code>sudo apt-get install build-essential linux-headers-generic linux-headers-`uname -r`</code>
téléchargez le paquet suivant :  \\
[[http://dl.dropbox.com/u/57056576/DRIVERS/REALTEK/rtl_92ce_92se_92de_8723ae_linux_mac80211_0006.0514.2012.tar.gz]] \\ \\
Extraire ce fichier tar.gz via clic droit et « Extraire ici ».\\ \\
Glisser/déposer le dossier décompacté sur le Bureau \\ \\
Se placer dans le dossier ou a été extrait le pilote puis tapez ceci dans le terminal : \\
Par exemple, si on l'a extrait sur le bureau on tapera (Par exemple, si on l'a extrait sur le bureau) : \\
<code>cd ~/Bureau/rtl_92ce_92se_92de_8723ae_linux_mac80211_0006.0514.2012/</code>
Installer le pilote en entrant successivement les commandes suivantes : \\
<code>make</code>
<code>sudo make install</code>
tester le pilote en chargeant son module via cette commande :  \\
<code>sudo modprobe rtl8723e</code>

=== Lecteur de carte SD non reconnu ===
Dans le terminal, coller ceci : lspci \\
si le résultat est : \\
<code>09:00.0 Unassigned class [ff00]: Realtek Semiconductor Co., Ltd. Device 5229 (rev 01)</code> 
ou (modèle Vigogne Ivy-Bridge fin 2012 par exemple) \\
<code>03:00.0 Unassigned class [ff00]: Realtek Semiconductor Co., Ltd. Device 5289 (rev 01)
</code>
téléchargez et installez le paquet debian : \\
[[http://planet76.com/drivers/realtek/rts-bpp-dkms_1.1_all.deb]]  \\
puis tapez ceci dans le terminal : \\
<code>sudo modprobe rts_bpp</code>
Puis insérer une carte SD dans le lecteur pour vérifier que cela fonctionne. Si cela fonctionne, collez ceci dans le terminal : \\
<code>echo 'DRIVERS=="rts_bpp", ENV{ID_DRIVE_FLASH_SD}="1"' | sudo tee -a /lib/udev/rules.d/81-udisks-realtek.rules</code> \\

==== Testé sous 10.04 et 12.04 LTS ====
=== Brother : Scanner non reconnu ===
téléchargez et installer le bon pilote sur le site de Brother [[http://welcome.solutions.brother.com/bsc/public_s/id/linux/en/download_scn.html]] \\
Éditez le fichier 40-basic-permissions.rules : \\
<code>sudo gedit /etc/udev/rules.d/40-basic-permissions.rules</code>
Repérez la section « USB devices » et remplacez la valeur « 0664 » par « 0666 » (elle apparait deux fois). \\
Vous devez donc avoir remplacé ceci : \\
<code># USB devices (usbfs replacement)
SUBSYSTEM=="usb", ENV{DEVTYPE}=="usb_device", MODE="0664"
SUBSYSTEM=="usb_device",        MODE="0664"</code>
par ceci : \\
<code># USB devices (usbfs replacement)
SUBSYSTEM=="usb", ENV{DEVTYPE}=="usb_device", MODE="0666"
SUBSYSTEM=="usb_device",        MODE="0666"</code>
Nota : il ce peut que le fichier soit vide, dans ce cas, coller les trois lignes ci-dessus puis enregistrer. \\
Ensuite, modifier le fichier 40-libsane.rules :  \\
<code>sudo gedit /lib/udev/rules.d/40-libsane.rules</code>
en rajoutant ceci : \\
<code># Brother scanners
ATTRS{idVendor}=="04f9", ENV{libsane_matched}="yes"</code>
Juste avant les lignes suivantes : \\
<code># The following rule will disable USB autosuspend for the device
ENV{libsane_matched}=="yes", RUN+="/bin/sh -c 'test -e /sys/$env{DEVPATH}/power/level && echo on > /sys/$env{DEVPATH}/power/level'"
LABEL="libsane_rules_end"</code>
Puis enregistrer le tout et redémarrer.
