==== Organisation des dossiers et sous dossiers - Clarification ====

Mise à jour de la page modèle avec les mot clés ubuntu/kubuntu et les versions en pages liées pour la génération automatique via backlink, d'index. Les admins proposent que cette page modèle soit affichée par defaut lors de la création d'une page

De même les différents titres de section ne se détachent pas suffisamment dans la page, et le h1 est clairement inutile en l'état puisque le titre de la page est affiché par Doku automatiquement --> suppression du bouton h1
La CSS sera modifiée en conséquence, pour prendre en compte aussi les difficultés de lecture que peuvent rencontrer les daltoniens

==== « Premiers pas sous ubuntu » ====

Un squelette pour les docs destinées aux débutants sera mis en place, ainsi qu'une page meta reprenant toutes les pages dont la doc est plus facile d'accès.

==== Ton des docs, que peut on se permettre ? ====

Statu quo, ton le plus neutre possible, même pour les pages sur les jeux.

==== Liste des relecteurs, optimisation des relectures, mise en place de codes pour simplifier les relectures, développement des taches liées ====

La création de page sera toujours ouverte à tous mais dans un dossier consacré : 'A classer' (terme à définir) ainsi les visiteurs pourront toujours créer des pages et l'équipe wiki s'occuppera de les placer au bon endroit. La page créée sera dans ce dossier tampon jusqu'à validation par l'équipe de relecture, relecture sur la grammaire et l'orthographe mais aussi sur la doc : reproduire/tester tant que possible les manips proposées
Un compte imap sera créé pour quelques relecteurs, le compte sera abonné à wiki changes, les modifications seront centralisées et les relecteurs se contenteront de vérifier les relectures non encore traitées par un autre relecteur (et pas chacun l'intégralité des modifications pour tous)
Mise en place d'une liste de mots clés décrivant la modification de la page, ainsi pas de temps perdu à détecter où a eu lieu le changement


==== Migration du wiki vers une autre solution ? retour d’expérience sur d’autres wikis ====

Suite au démontage en règle de mediawiki par jd_, la possibilité d'une migration vers mediawiki est completement oubliée

==== Plugins à mettre en place ====

  - Category : Les tests sur le plugins tags ne sont pas très concluants, le besoin d'avoir un gestion de tags propre dans Doku ne doit pas être spécifique à ubuntu-fr, Yann contacte l'équipe de Doku
  - Discussion : l'utilisation du forum comme espace de discussion pour chaque page du wiki étant condamnée (intégration trop forte, un outil pour un besoin) le plugin de doku Discussion sera testé plus loin
  - Un cron qui pour chaque mot clé génèrera la page index correspondante est à l'étude  

==== Authentification ====

Le systeme d'authentification est très efficace et soulage pas mal le travail de veille des relecteurs contre le vandalisme, dans le meme ordre d'idée les pages meta et surtout l'accueil du site seront éditables pour les seuls utilisateurs authentifiés

==== Quels docs traduire/faire ? ex dvdshrink soft non libre sous win... Quelle latitude sur les docs ? Panda vs Clam, etc. ====

Les docs traitant de soft non-libres sont toujours à l'ordre du jour par contre une page dédiée sur le concept propriétaire/libre sera astucieusement placée dans l'en-tête des docs des softs non libres
